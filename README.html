The Problem
Why this exists
During a 7:45–9:15 AM rush, a skilled barista makes dozens of implicit decisions: pulling two espressos simultaneously, steaming milk while shots extract. An untrained barista serializes everything. The difference is 2–3 minutes per order. With 40 orders in rush hour, that is 80+ minutes of lost throughput daily. No existing tool tells your team what to do next, in real time.

Core Modules
Five ML components, each independently testable and deployable.
☕
Complexity Scorer
Scores every drink 0–10 across 5 features: extraction method, milk type, customizations, temperature, and shot count. Powder-prep drinks like matcha get a bonus step for whisking. Powers slot assignment decisions.
⚡
Concurrent Scheduler
Greedy algorithm assigns drinks to machine slots in priority order, maximising parallel execution. No two drinks ever collide on the same machine. Produces a live "Start now + While that pulls" action pair.
🧠
RL Feedback Loop
Tabular Q-learning agent updates policy weights with every order completion. Reward shaped across throughput gain, wait reduction, concurrency bonus, and error penalty. Checkpoint save/load included.
📡
POS Stream
WebSocket listener normalises real POS payloads. In demo mode, a mock stream generates orders on a realistic rush-hour demand curve with configurable rate, multiplier, and error injection.
📊
Demand Forecaster
Time-of-day, day-of-week, and weather signals feed a demand curve that predicts volume 15 minutes ahead. Powers rush-mode auto-activation and pre-warm recommendations for the next morning.
🖥
Barista Tablet UI
Streamlit dashboard with parchment-and-espresso premium design. Shows optimized queue, machine state, live RL signals, POS stream log, and a post-rush report with throughput vs baseline chart.

Quick Start
Up in under 2 minutes
Clone, install, run. No config required in demo mode.
1
Clone and enter the project
git clone https://github.com/YOUR_USERNAME/barista-iq && cd barista-iq
2
Create a virtual environment and install dependencies
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
3
Run the app
streamlit run app/main.py
Opens at localhost:8501. Uses demo mode by default — no POS connection needed.
4
Run the test suite
pytest tests/
52 tests. Should all pass on Python 3.10+.
5
Optional: start the API alongside the UI
docker-compose up
Streamlit at :8501 · FastAPI docs at :8000/docs
